Exceptions on the stack

You are given a description of the current state of the program. Up to this point, we’ve made the chain of function calls and the latest called function currently handles the exception. You are given the stack of functions calls, the behaviour of the functions when handling various exceptions and the name of the currently processed exception.

Functions can operate with an exception as follows.

  • The function does not handle (does not catch) the exception. In this case, the function is removed from the call stack, and the exception is thrown further.
  • The function handles the exception and throws a new exception. In this case, the function is removed from the call stack and the new exception is thrown further.
  • The function handles the exception and continues to work correctly. In this case, the function remains on the stack of calls and no new exceptions are generated.

Input data format

The first line of input data contains number n – the size of the call stack (1 ≤ n ≤ 40).
The second line of input data contains n lines, separated by a space – the names of the functions, located on the stack, in the order they were called.
The next line contains number m (1 ≤ m ≤ 100).
The following m lines contain the description of the functions behaviour when handling exceptions.
A description of the behavior is as below.
<function> <handled exception> <throwed exception>.
For example, fun e1 e2. This entry indicates that when processing the e1 exception, the fun function will generate the exception e2.
The underscore symbol '_' may be present instead of the thrown exception name. This means that when processing the exception, the functions continues to operate properly and does not generate any new exceptions.
The name of the function or the exception - the string consisting of the lowercase letters of the Latin alphabet, having the length of not more than 5.

The last line of the input contains the name of the currently processed exception.

Output data format

In the single line output the names of the functions, which will remain on the call stack after processing all the exceptions in the order, in which they were called. In the case if there are no functions left on the stack, output an empty line.


Sample Input:
3
a b c
4
c ex1 ex2
b ex3 _
a ex2 _
c ex3 ex3
ex1

Sample Output:
a
