On the input, the program is given two matrices, each in the following format: the first line contains two positive integers \( n \) and \( m \) separated by a space - matrix dimensions. The next line contains \( n⋅m \) space separated integers - elements of the matrix. It is assumed that the matrix is filled line-by-line, which means the first \( m \) numbers – the first row of the matrix, the numbers from \( m+1 \) to \( 2m \) - the second row, etc.

Print the product of matrices \( XY^T \), if their forms match, otherwise print the line "matrix shapes do not match" (if the forms of the matrices do not match properly).

At this time, we have done the preparatory work on reading matrices for you (when you start solving, this code will be already written):

x_shape = tuple(map(int, input().split()))
X = np.fromiter(map(int, input().split()), np.int).reshape(x_shape)

A few comments:

  • map(f, utterable, …) — built-in Python function, returns the result of by-element application of the function f to the elements of the iterablesequence; if f takes several arguments, there should be the same number of sequences: the result will be presented by the map(f, x, y, z) iterator, subsequently returning f(x[0], y[0], z[0]), f(x[1], y[1], z[1]), f(x[2], y[2], z[2]) and so on; the result of the applying f to the next set of arguments will be calculated only when you need to use this result, but not earlier. Please read the shorted and more detailed description here;
  • np.fromiter creates the NumPy-array from the iterator - it get all values from the iterator and stores them in the array;
  • input() — built-in Python function; reads one line (a sequence of characters until the line break symbol) from the input data stream and returns it as a string
  • split() — the methods of the stringclass; returns the list of words in the string (here words are the sequences of symbols, separated by spaces); allows additional arguments; please read more in documentation.
