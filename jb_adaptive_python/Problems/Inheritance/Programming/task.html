You are given the description of class inheritances in the following format.
<class name 1> : <class name 2> <class name 3> ... <class name k>
It means that class 1 is inherited from class 2, clacc 3, etc.

It is equivalent to the following class declaration in Python:

class Class1(Class2, Class3 ... ClassK):
    pass

Class A is an immediate ancestor of class B, if B is inherited from A:

class B(A):
    pass

Class A is an ancestor of class B, if one of the following:

  • A = B;
  • A - immediate ancestor of B
  • there is such class C that C is an immediate ancestor of B and A - ancestor of C
For example: 

class B(A):
    pass
class C(B):
    pass

You should answer the queries whether one class is the ancestor of another class

Important note:

It is not required to create classes.

We ask you just to model this process and understand where there is a path from one class to another.

Input data format

In the first line contains the integer n - the number of classes.

The next n lines contain the description of the class inheritances. The i-th line specifies from which classes this i-th class is inherited. Please note that the class can be not inherited. It is guaranteed that the class is not inherited from itself (directly or indirectly), that the class is not inherited from another class for more than once.

The next line contains number q - the amount of queries.

The following q lines contain the description of these queries in the following format: <class name 1> <class name 2>.

The class name is a string consisting of the characters of the Latin alphabet, having the length of not more than 50 characters.

Output data format

For each of the queries output in a separate line the word "Yes", if class 1 is an ancestor of class 2, and "No", if it is not.


Sample Input:
4
A
B : A
C : A
D : B C
4
A B
B D
C D
D A

Sample Output:
Yes
Yes
Yes
No
