You are given a description of the exception classes inheritance in the following format.
<exception name 1> : <exception name 2> <exception name 3> ... <exception name k>
It means that exception 1 is inherited from exception 2, exception 3, etc.

Or equivalent to the entry:

class Error1(Error2, Error3 ... ErrorK):
    pass

Anthony wrote the code that looks as follows.

try:
   foo()
except <name 1>:
   print("<name 1>")
except <name 2>:
   print("<name 2>")
...

Kyle looked at this code and told Anthony that he may not catch some of the exceptions, as earlier their ancestor may be caught in the code. But Anthony does not remember what exceptions are inherited from what. Help him to escape this awkward situation. Write a program that will find which exception handlings can be removed from the code.

Important note:

The types of exceptions are not created. There is no need to create classes of exceptions either. Please model this process and understand which exceptions don't need to be called, because we have previously caught their ancestors.

Input data format

First line of the input data contains integer n - the number of exception classes.

The next n lines contain the description of the class inheritance. The i-th line specified from which classes the i-th class is inherited. Please note that a class may not inherit from anything. It is guaranteed that the class does not inherit from itself (directly or indirectly), that the class does not inherit explicitly from one class more than once.

The next line contains number m - the number of exceptions to handle.
Next m lines contain the names of exceptions in the order, in which they were written in the Anthony's code.
It is guaranteed that no exception is handled twice.

Output data format

In a separate line output a name of each exception, the handling of which can be removed from the code without changing the behaviour of the program. Output these  names in the same order in which they were in the input data.

Test example 1

Let’s check the code:

try:
   foo()
except ZeroDivision :
   print("ZeroDivision")
except OSError:
   print("OSError")
except ArithmeticError:
   print("ArithmeticError")
except FileNotFoundError:
   print("FileNotFoundError")
...


Kyle looked at this code and told Anthony that he may not catch the exception FileNotFoundError, because we already catch OSError – the ancestor of FileNotFoundError.


Sample Input:
4
ArithmeticError
ZeroDivisionError : ArithmeticError
OSError
FileNotFoundError : OSError
4
ZeroDivisionError
OSError
ArithmeticError
FileNotFoundError

Sample Output:
FileNotFoundError
